# 基于 YOLO26 的低空无人机视觉检测基线系统构建与实验报告

## 摘要

为支撑后续毫米波雷达与视觉融合探测任务，本文先独立于 `mmradar` 主工程搭建了一条可复现、可训练、可评估、可导出、可备份的视觉检测基线。系统以 `Anti-UAV300` RGB 数据为主数据源，采用 `YOLO26` 系列模型作为检测骨干，在本地完成数据清洗、格式转换、冒烟训练和训练脚本链路验证后，将正式训练迁移至 A800 训练服务器执行。在 `YOLO26s + imgsz=960 + batch=64` 的设置下，模型于 A800 上完成正式训练并触发早停，最佳模型出现在第 18 个 epoch，最终验证集指标达到 `precision≈0.980`、`recall≈0.978`、`mAP50≈0.993`、`mAP50-95≈0.686`；在独立测试集上达到 `precision≈0.961`、`recall≈0.900`、`mAP50≈0.937`、`mAP50-95≈0.537`。在此基础上，本文还完成了三段测试视频离线推理、`predictions.jsonl` 结构化结果输出、ONNX 导出与 `onnxruntime` 最小验证，并将关键训练结果、日志与模型资产完整备份到本地。实验表明，该视觉分支已经具备作为后续多传感器融合前端感知模块的工程基础。

## 关键词

低空无人机检测；YOLO26；Anti-UAV300；视觉基线；目标检测；ONNX 导出

## 1 引言

低空小目标无人机在远距离、复杂背景和纯天空场景下具有明显的小目标特性，仅依靠单一传感器往往难以兼顾探测距离、定位精度与误检控制。考虑到后续系统目标是开展毫米波雷达与视觉融合探测，因此在融合之前，有必要先建立一条独立、稳定、可复现实验结果的视觉检测链路。

本文工作的核心目标不是直接进入融合实现，而是先回答以下三个问题：

1. 是否可以在现有工作区内独立搭建一条可维护的视觉分支；
2. 是否可以在公开反无人机数据集上训练出一套有实际参考价值的正式视觉基线；
3. 是否可以将训练、评估、推理、导出和备份链路全部打通，为后续报告撰写和融合接口设计提供稳定基础。

围绕上述目标，本文完成了从数据准备、训练环境搭建、正式训练迁移、结果验证、模型导出到资产归档的一整套工程化工作。

## 2 系统构建与方法

### 2.1 工作区组织

视觉分支在仓库根目录下以 `vision_uav/` 独立组织，不直接嵌入 `mmradar` Qt 主程序。其目录划分如下：

- `configs/`：集中保存训练、评估、推理、导出配置；
- `scripts/`：集中保存数据准备、训练、评估、推理、导出与环境检查脚本；
- `docs/`：保存运行说明、数据说明、结果说明和后续报告材料；
- `data/`：保存原始数据与处理后数据；
- `runs/`：保存训练、评估、推理与备份结果；
- `weights/`：保存预训练权重。

这种分层方式使视觉链路与 `mmradar` 的雷达链路在工程上解耦，便于后续单独演进与再集成。

### 2.2 数据准备方法

本工作选择 `Anti-UAV300` 的 RGB 数据作为主训练集。原因在于该数据集面向反无人机场景，目标尺度小、背景复杂，更接近本课题后续的低空无人机探测需求。

数据处理流程包括：

1. 下载并解压 `Anti-UAV300`；
2. 解析其 `visible.mp4 + visible.json` 序列标注结构；
3. 按视频序列而非按帧划分 `train/val/test`；
4. 转换为 YOLO detect 格式；
5. 保留空标签帧作为背景样本；
6. 在 Windows 中文路径环境下引入 ASCII alias 以保证 `Ultralytics` 工具链正常运行。

处理后的正式数据集规模如表 1 所示。

| 数据划分 | 序列数 | 帧数 | 正样本帧数 | 空标签背景帧数 |
| --- | ---: | ---: | ---: | ---: |
| Train | 160 | 29,924 | 28,470 | 1,454 |
| Val | 67 | 6,208 | 5,850 | 358 |
| Test | 91 | 8,547 | 7,992 | 555 |

表 1 正式检测数据集统计

### 2.3 模型与训练链路

本工作分两个阶段搭建模型训练链路。

第一阶段在本地 4070 环境上进行冒烟验证，使用 `YOLO26n` 进行 1 epoch 小规模训练，主要用于确认：

- 数据转换结果可被训练器正确读取；
- CUDA 环境与 `Ultralytics` 调用链可正常工作；
- 训练、评估、推理和导出脚本入口均可被独立调用。

第二阶段在 A800 服务器上进行正式训练。考虑到 A800 的显存容量远大于本地 4070，训练配置切换为：

- 模型：`YOLO26s`
- 输入尺寸：`960`
- Batch size：`64`
- 训练轮数上限：`30`
- 早停容忍：`8`
- `cache=ram`
- `workers=16`

在训练实现上，本文补齐了如下正式入口：

- 训练脚本：`train.py`
- 独立测试集评估脚本：`evaluate.py`
- 离线视频推理脚本：`infer_video.py`
- ONNX 导出脚本：`export_onnx.py`
- 硬负样本评估脚本：`evaluate_hard_negatives.py`

### 2.4 结果输出协议

为兼容后续与雷达链路的对接，本文统一了视觉检测结果的结构化输出格式。每段视频推理均输出：

- `overlay.mp4`：叠加检测框后的可视化视频；
- `predictions.jsonl`：逐帧结构化检测结果。

其中每帧结果至少包含：

- `source_id`
- `frame_id`
- `timestamp_ms`
- `width`
- `height`
- `detections`

每个检测项至少包含：

- `bbox_xyxy`
- `class_id`
- `class_name`
- `confidence`

这使得视觉结果既可以直接用于人工分析，也为后续与雷达时间戳对齐和目标级融合留下了明确接口。

## 3 实验环境与实现过程

### 3.1 本地环境

本地环境主要用于数据准备、脚本开发和小规模冒烟验证。关键环境如下：

- GPU：`NVIDIA GeForce RTX 4070 Laptop GPU`
- 独立 Conda 环境：`vision-uav`
- Python：`3.11.15`
- `ultralytics`：`8.4.37`

本地环境的主要价值在于快速迭代代码与验证训练链路是否可启动，而非承担全量正式训练。

### 3.2 A800 远端训练环境

正式训练、测试评估、三段视频推理与 ONNX 导出均在远端 A800 机器上完成。由于远端访问 `GitHub` 和 `Hugging Face` 受限，本工作额外完成了如下工程处理：

1. 使用 SSH 将本地仓库和处理后数据同步到远端；
2. 通过远端代理脚本解决权重下载与依赖安装问题；
3. 修正 Linux 训练机上的路径兼容问题；
4. 将正式训练、评估、推理和导出日志全部独立保存。

## 4 实验结果与分析

### 4.1 验证集结果

正式模型 `best.pt` 的验证集结果如表 2 所示。

| 指标 | 数值 |
| --- | ---: |
| Precision | 0.980 |
| Recall | 0.978 |
| mAP50 | 0.993 |
| mAP50-95 | 0.686 |

表 2 正式模型在验证集上的结果

结合 `results.csv`，最佳行对应：

- `epoch = 18`
- `precision = 0.97998`
- `recall = 0.97897`
- `mAP50 = 0.99289`
- `mAP50-95 = 0.68635`

从结果上看，该模型在验证集上已达到较高的检测精度和召回率，说明训练链路和数据准备流程整体有效。

### 4.2 测试集结果

为避免将 `val` 结果误当作泛化能力，本文进一步在 `test` 集上进行了独立评估，结果如表 3 所示。

| --- | ---: |
| Precision | 0.96060 |
| Recall | 0.90000 |
| mAP50 | 0.93655 |

测试集结果低于验证集，尤其 `mAP50-95` 下降较明显，这说明模型虽然在主数据集上已经具备良好识别能力，但在未参与调参的数据上仍然存在一定的定位精度退化与泛化损失。这也为下一阶段的误检分析和 hard negative 补强提供了明确方向。

## 6 讨论

从当前结果来看，视觉分支已经取得了两个关键成果。

第一，工程闭环已经打通。数据准备、训练、评估、推理、ONNX 导出和本地备份都已完整完成，后续任何实验都不需要再从零搭建基础设施。

第二，模型已具备可用的正式基线性能。验证集与测试集结果表明，该模型在公开反无人机数据上已经具备较强检测能力，可以作为后续融合实验的视觉参考分支。

同时，当前结果也暴露出两个明确的后续问题：

1. `test mAP50-95` 明显低于 `val mAP50-95`，说明模型在更严格定位精度指标上仍存在泛化损失；
2. 当前尚未对鸟类视频或纯天空背景做系统性的 hard negative 误检分析，因此还不能判断“鸟类是否是主要误检来源”。

因此，下一阶段不应盲目重复正式训练，而应转向：

- 错误分析；
- 硬负样本误检评估；
- 与 `mmradar` 时间戳、目标级输出的接口对接准备。

## 7 结论

本文围绕低空无人机视觉检测分支，完成了一条独立于 `mmradar` 主程序的 YOLO26 视觉基线系统构建工作。通过 `Anti-UAV300` 数据准备、本地冒烟验证、A800 正式训练、独立测试集评估、三段视频推理、ONNX 导出与本地双阶段备份，本文已经建立了一套可复现、可复核、可继续扩展的视觉检测基础。

结果表明，当前正式模型在验证集上达到 `precision≈0.980`、`recall≈0.978`、`mAP50≈0.993`、`mAP50-95≈0.686`，在测试集上达到 `precision≈0.961`、`recall≈0.900`、`mAP50≈0.937`、`mAP50-95≈0.537`。该结果已经能够支撑后续开展错误分析、鸟类硬负样本评估以及视觉-雷达融合接口设计。
